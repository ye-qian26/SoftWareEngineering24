# 用户对话隔离功能实现计划

## 目标
实现用户登录/注册功能，让每个用户有独立的对话记录，不同用户之间无法看到对方的会话。

## 当前状态分析

### 后端已有
- `user` 表（id, username, created_at）
- `conversation` 表（id, user_id, title, created_at, updated_at）
- `message` 表（id, conversation_id, role, content, created_at）
- 根据 userId 查询会话列表的接口

### 前端已有
- userId 写死为 1：`const [userId] = useState(1)`

## 实现步骤

### 第一步：后端 - 用户模块

1. **创建 UserDTO**
   - 新建 `UserDTO.java`
   - 包含：id, username, password 字段

2. **创建登录/注册请求 DTO**
   - 新建 `LoginRequest.java`（username, password）
   - 新建 `RegisterRequest.java`（username, password）

3. **创建 Token 响应 DTO**
   - 新建 `TokenResponse.java`（token, userId, username, expiresIn）

4. **实现用户注册接口**
   - POST `/api/user/register`
   - 验证用户名是否存在
   - 密码加密存储（使用 BCrypt）

5. **实现用户登录接口**
   - POST `/api/user/login`
   - 验证用户名和密码
   - 生成 JWT Token

6. **创建 JWT 工具类**
   - 新建 `JwtUtil.java`
   - 生成 Token
   - 验证 Token
   - 解析用户信息

7. **创建 JWT 过滤器**
   - 新建 `JwtAuthFilter.java`
   - 拦截请求，验证 Token
   - 将用户信息放入请求属性

8. **修改 ConversationService**
   - 新增方法：根据 conversationId 和 userId 验证会话归属
   - 防止用户访问其他用户的会话

9. **修改 ChatService**
   - 新增方法：验证消息所属用户

10. **修改 MessageMapper**
    - 新增：根据 conversationId 查询时验证 userId

### 第二步：数据库更新

1. **更新 user 表**
   - 添加 password 字段（存储加密后的密码）
   ```sql
   ALTER TABLE `user` ADD COLUMN `password` VARCHAR(255) NOT NULL COMMENT '密码';
   ```

### 第三步：前端 - 用户模块

1. **创建登录页面组件**
   - 新建 `src/pages/Login.jsx`
   - 用户名、密码输入
   - 登录/注册切换按钮
   - 登录成功后保存 token 和 userId

2. **创建注册页面组件**
   - 新建 `src/pages/Register.jsx`
   - 用户名、密码、确认密码输入
   - 注册成功后跳转到登录页

3. **修改 App.js**
   - 添加路由（登录页/注册页/主聊天页）
   - 添加 token 验证逻辑
   - 未登录时显示登录页
   - 登录后从 localStorage 读取 userId

4. **创建用户 API 接口**
   - 新建 `src/api/user.js`
   - login(username, password)
   - register(username, password)

5. **修改现有 API 拦截器**
   - 为请求自动添加 Authorization Header
   - 处理 token 过期

6. **修改 ConversationList 组件**
   - 加载当前用户的会话列表

### 第四步：测试验证

1. 测试用户注册
2. 测试用户登录
3. 测试 Token 认证
4. 测试不同用户的会话隔离
5. 测试未登录无法访问聊天
